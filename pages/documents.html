Typography No Print Users can add a ‘printOnly’ class to areas that will only show when printed. Forms The forms in Fresh Plate are standard HTML/CSS forms with some enhancements. Below is a list of classes that format various objects such as text inputs, buttons, fieldsets, labels, etc. Be sure and check with your Hosting provider as forms may require scripts. Included with the download is a contactForm.html and ThankYou.html that may be used as guides to creating your own forms. They are in the plugin directory. They are set up to work on an ipage.com supplied website. You will need to change the script path and the email address in the input hidden field as well as verify the path to Thankyou.html. Note: the example forms in the example pages do not work. No eMail or Data is sent. Grid System The grid system in FreshPlate framework is based on 960px wide grid - 12 column grid solution. Each column width is 100% / 12 = 8.33%. All rows should be wrapped in a div . All columns should be wrapped in a 'row' and should add up to 12. The grid acts responsively with the following breakpoints: Mobile portrait (Default) - Single column. No need to define. But for those users who want more granular control of mobile screen you can use classes to further split your container. LARGE SCREEN TABLET VIEW MOBILE LANDSCAPE MOBILE PORTRAIT Tables Tables are borderless by default except for a bottom border on all (table headings) You can add the following classes to tables: ‘border’ makes a solid gray border of 1px thick all around ‘striped-even’ or ‘striped-odd’ to create striping (Note: green stripes is just an example to make it stand out… ) ‘Center’ to center the table ‘hover-row’ to be able to highlight a row when cursor mouses over ‘hover-cell’ to be able to highlight a cell when cursor mouses over Buttons .btnStd - Standard Button – No class applied. Includes Hover and Active States. .btnFlat – A standard, square transparent button. It will pick up the fill for the container it is located in. And is outlined with a 2-pixel border. .btnFlat-rnd-crnr – A Square transparent button. The background color inherited from the containing object. Border is 2-pixels, the border radius is .5em. .btnFlat-rnd-crnr – A 2d effect button with rounded corners (.5em), Inherits the color of the containing object. Hover and Active state enabled. .btn3d - A standard 3d button. Borders are 10-pixels. Has Active and Ho9ver state built in. .btn3d-rnd – A 3d oval shaped button with 50% radius corners. Borders are 10-pixels. Active and Hover state included. .btn3d-rnd-crnr – A 3d Button with rounded corners. Border Radius is.5em and border is 5-pixels. Active and Hover State included. Button Sizes The following class can be added to buttons to size them specifically. .btnTinySq – Small button with small font 60 pixels square font 7.5pixels .btnMed – width 120 Pixels by 40 Pixels Font 12 pixels .btnLrg – Width 160 Pixels by 70 Pixels, Fonts 16 pixels .btnHuge – Width 50% of container, Height: 70 Pixels, font 22pixels Responsive Images To make images responsive, use the class ‘responsiveImg’ Helper Classes Icons The default icon set is Material Design Icons.
